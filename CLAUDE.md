\# 비문 신청서 시스템 (CLAUDE.md)



\## 1. 목적



비문 신청서를 입력받아

양식별 규칙에 맞게 글자수를 계산하고 금액을 산출하며

저장 및 불러오기 기능을 제공하는 웹 시스템



\---



\## 2. 전체 구조



\### 기본 흐름



1\. 대분류(category) 선택

2\. 세부유형(subcategory) 선택

3\. 양식 규칙 자동 적용

4\. 전면 / 측면 / 후면 입력

5\. 글자수 계산

6\. 금액 계산

7\. 저장 및 불러오기



\---



\## 3. 양식 구조



\### 대분류 / 세부유형



\* 매장묘: 단장, 쌍분

\* 단납/평장1기: 기본

\* 쌍납/평장부부A: 기본

\* 송수재: 개인, 부부

\* 부부A: 기본

\* 납골정: 기본

\* 평장부부B: 기본, 1번, 2번, 3번, 4번

\* 평장1기P: 기본

\* 4기: A, B, C, P, R

\* 평장부부P: 기본

\* 평장4기: 기본

\* 평장4기P: 기본

\* 8기: 기본

\* 다기형: 12A, 20기, 30기

\* 12기B: 기본

\* 24기: 기본

\## 5. 입력 구조



\### 전면



\* 이름: 큰글씨

\* 날짜: 작은글씨

\* 세례명 및 기타: 큰글씨

\* 종교마크: 1자 처리



\### 측면



\* 생년월일: 작은글씨

\* 사망일 + 구분: 작은글씨



\### 후면



\* 관계 문구: 작은글씨

\* 기타 문구 포함



\---



\## 6. 계산 규칙



\### 기본



\* 공백, 줄바꿈 제외하고 글자수 계산

\* 큰글씨 / 작은글씨 분리 계산



\### 매장묘 (maejang)



\* 지묘 2자 고정

\* 남자: 이름 + 본관 + 호칭 + "공" 1자 추가

\* 여자: 이름 + 본관 + 호칭 + "씨" 1자 추가

\* 측면 날짜 따로 계산



\### 일반형 (standard)



\* 슬롯 수에 따라 반복 입력

\* 가족지묘 문구 존재 시 추가 계산



\### 후면



\* 관계별 접두어 길이 포함하여 계산



\---



\## 7. 금액 계산



\* 글자수 × 단가

\* 전면 / 측면 / 후면 각각 계산

\* 스톤포토 조건 시 120000 추가

\* 평장4기 + 스톤포토 → 후면 한글 고정
