# 비문신청서 프로젝트 작업 규칙

## 🔒 절대 규칙

* 전체 코드 재작성 금지
* 기존 기능(계산 / 저장 / 불러오기 / 프린트) 절대 수정 금지
* 요청한 부분만 최소 수정
* 공통 미리보기 구조 절대 재작성 금지
* 기존 레이아웃/DOM 구조 유지

\---

## 🧭 작업 절차 (반드시 지킬 것)

1. 코드 수정 전:

   * 변경될 파일명
   * 함수명 / 템플릿 위치
먼저 출력
2. 사용자 확인 후:

   * 해당 파일/위치만 수정 진행

\---

## 🛑 금지 사항 (중요)

* 전체 파일 재작성 금지
* unrelated 코드 수정 금지
* 리팩토링 금지
* 구조 변경 금지
* 컴포넌트 분리 금지
* 함수 새로 만들기 최소화

\---

## 🎨 출력 / 레이아웃 규칙

### 공통

* 가운데 정렬 금지
* 기존 정렬 방식 유지
* 불필요한 스타일 추가 금지

\---

## ✍️ 텍스트 처리 규칙

### 자손 이름

* 글자 쪼개기 금지
* split / map 사용 금지
* reverse 금지
* join(" ") 금지
* textarea 줄바꿈 그대로 유지

\---

### 자손 항목명

* 글자 단위 세로 출력 허용
* 단, 이름에는 절대 적용 금지

\---

### 날짜 출력

* 숫자 한 글자씩 출력
* 점(.) 포함
* 공백 제거
* letter-spacing 사용 금지
* transform 사용 금지

\---

## 🏗️ 구조 유지 규칙

### 매장묘 / 평장 구조

* 서로 다른 레이아웃 절대 섞지 말 것
* 각 시안별 구조 유지

\---

### 매장묘 후면 자손

* 상단 / 하단 구조 유지
* "손"은 항상 "자" 아래
* "외손"은 항상 "녀" 아래
* 세로 라인 정렬 유지

\---

### 평장1기 / 단납 후면 자손

* 라벨 + 이름 baseline 정렬
* 라벨 width 고정
* 두 번째 줄부터 들여쓰기 유지
* 공백 trim 금지

\---

## ⚙️ 스타일 수정 규칙

* 기존 CSS 클래스 우선 수정
* inline 스타일 남용 금지
* 새로운 스타일 추가 최소화

\---

## 📌 기타

* 기존 변수명 유지
* 기존 함수명 유지
* 기존 데이터 흐름 유지
